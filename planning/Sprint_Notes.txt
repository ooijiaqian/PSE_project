Project: HydrateMate
Methodology: Agile Scrum

🔵 Sprint 1 – Planning & Design Phase
Sprint ID: SPR001

Sprint Goal
To establish the foundation of the project by defining the product vision and creating system design models.

Sprint Planning
During Sprint Planning, the team reviewed the Product Vision and initial user stories. The high-level requirements were transformed into design tasks.
User stories were analyzed and converted into modelling tasks to prepare system structure before development.

Sprint Backlog Tasks
-Define Product Vision
-Create Use Case Diagram
-Create Class Diagram
-Design Low Fidelity Wireframes

Implementation Approach
1. Product Vision
- The team refined the vision, target users, business goals, and key features to align with SDG 3.

2. Use Case Diagram
- User stories were analyzed and grouped into system interactions. Main actor identified: User.
Core use cases such as login, log water, set goal, and view history were defined.

3. Class Diagram
-Classes such as User, WaterLog, Badge, Notification were identified from the use cases.
Attributes and relationships were defined.

4. Low Fidelity Wireframes
-Basic screen sketches were created to visualize layout and navigation flow.

Sprint Review
Completed:
✔ Product Vision finalized
✔ Use Case Diagram completed
✔ Class Diagram completed
✔ Low-fidelity wireframes created

Sprint Retrospective

What went well:
Clear task division
Good communication

What can be improved:
More detailed discussion before diagram creation

🔵 Sprint 2 – Core Feature Design
Sprint ID: SPR002

Sprint Goal
To design and implement core functional features in high-fidelity format.

Sprint Planning
User stories related to core hydration tracking were prioritized.
Requirements were converted into specific UI and prototype tasks.

Sprint Backlog Tasks
-Design Login Page
-Create High Fidelity Wireframes
-Implement One Tap Water Logging
-Implement Set Daily Goal Feature
-Implement Cup Size Customization
-Design Progress Display

Implementation Approach
Each feature was developed based on the functional requirements defined earlier.

Example:
FR1 → Login system
FR2 → One Tap Water Logging
FR3 → Set Daily Goal
High-fidelity wireframes were created using Figma to simulate real app experience.

Sprint Review
Completed:
✔ Login screen designed
✔ Core tracking features implemented in prototype
✔ Progress bar visualized

Sprint Retrospective
What went well:
Clear mapping from requirement to feature

What can be improved:
Time estimation for design tasks
